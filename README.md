这是一个可直接导入微信开发者工具的完整工程，已经按你的需求做成：

## 一、你拿到工程后应该先做什么

### 1）导入工程

用微信开发者工具打开本项目根目录：

```bash

```

导入时：

- `AppID` 填你自己的小程序 AppID
- 如果先调试界面，也可以先用测试号

### 3）安装并上传云函数依赖

在微信开发者工具里，对每个云函数执行：

- 右键云函数目录
- 选择“**在终端中打开**”
- 执行：

```bash
npm install
```

然后分别上传部署：

- `login`
- `userManager`
- `goodsManager`
- `messageManager`
- `setup`

### 4）首次运行 `setup` 云函数

部署完成后，先测试执行一次 `setup` 云函数，它会自动尝试创建这 5 个集合：

- `users`
- `goods`
- `conversations`
- `messages`
- `school_auth_requests`

如果集合已经存在，它会返回 `exists-or-failed`，这属于正常情况。

## 二、云开发数据库建议

### 集合说明

#### 1. `users`
保存用户资料：

- 昵称 `nickName`
- 头像 `avatarUrl`
- 个性签名 `signature`
- 用户手动填写的微信号 `wechatNumber`
- 学校认证状态 `schoolVerifyStatus`
- 是否已认证 `schoolVerified`

#### 2. `goods`
保存商品信息：

- 商品名称
- 描述
- 价格
- 分类
- 标签
- 图片 fileID
- 面交地点与地址
- 卖家 openid / 昵称 / 头像 / 微信号（创建商品时冻结一份）
- 状态：`on / sold / off`

#### 3. `conversations`
保存会话摘要：

- 该会话对应哪个商品 `goodsId`
- 卖家 openid `sellerOpenid`
- 买家 openid `buyerOpenid`
- 最近一条消息 `lastMessage`
- 最近消息时间 `lastMessageAt`
- 未读计数 `unreadMap`
- 商品快照 `goodsSnapshot`

#### 4. `messages`
保存每一条聊天消息：

- 所属会话 `conversationId`
- 发送方 / 接收方
- 文本内容
- 是否已读
- 时间

#### 5. `school_auth_requests`
保存学校认证申请：

- 真实姓名
- 学号
- 学院
- 专业
- 学生证图片
- 审核状态

### 权限建议

由于本项目**主要通过云函数读写数据库**，建议你把这几个集合都设置成更严格的权限，例如：

- **仅管理员可读写**

这样更安全，前端用户无法直接越权读取消息或用户资料。

## 三、页面功能说明

## 1. 首页 `pages/home`

### 已实现功能

- 顶部欢迎语：自动读取当前用户昵称，没有则显示“同学”
- 搜索框：按商品名称 / 标签 / 地点 / 描述做本地匹配
- 分类标签：按类别筛选商品
- 商品卡片：封面、标题、价格、分类、标签、地点、卖家头像昵称、发布时间
- 悬浮发布按钮

### 核心逻辑

- 页面通过 `goodsManager -> list` 拉取全部在售商品
- 再在前端本地做搜索与筛选
- 这样搜索体验更丝滑，不会每输一个字就请求云函数

---

## 2. 发布页 `pages/publish`

### 已实现功能

- 拍照 / 相册选图
- 上传到云存储
- 自动识别商品名称、分类、标签（优先 CloudBase AI）
- 用户可手动修改 AI 结果
- 选择标签
- 填写描述
- 填写期望价格
- 选择校内位置
- 发布到 `goods` 集合

### 自动识别原理

发布页会：

1. 先把第一张商品图上传到云存储
2. 通过 `wx.cloud.getTempFileURL` 拿临时地址
3. 调用 `wx.cloud.extend.AI` 的混元视觉模型 `hunyuan-vision`
4. 让模型输出 JSON：
   - `title`
   - `category`
   - `tags`
   - `reason`
5. 页面把识别结果自动回填到表单

### 重要说明

如果你的环境还没有配置 AI，大模型调用会失败，但页面**不会报废**，会自动降级成**手动填写模式**。

也就是说：

- 有 AI：自动识别 + 手动修正
- 没 AI：纯手动填写

## 3. 私信模块 `pages/message` + `pages/chat`

### 已实现功能

- 会话列表
- 最近消息预览
- 未读数显示
- 进入聊天详情页
- 聊天详情页轮询拉取消息
- 发送文字消息
- 会话关联商品快照
- 若对方填写了微信号，可一键复制

### 私信原理

#### 会话创建逻辑

用户在商品详情页点击“私信联系卖家”时：

- 调用 `messageManager -> ensureConversation`
- 如果当前用户和卖家在这件商品下已经聊过，直接复用旧会话
- 如果没有，则创建一个新会话

#### 消息发送逻辑

发消息时：

- 新增一条 `messages` 文档
- 同时更新 `conversations`：
  - `lastMessage`
  - `lastMessageAt`
  - `unreadMap`

#### 为什么用了轮询而不是实时监听

为了让你导入后更稳、更容易调试，这个版本先用 **3 秒轮询**：

- 优点：实现简单、稳定、可控
- 后续如果你想做成更丝滑的“实时聊天”，可以升级为数据库 `watch`

## 4. 我的页 `pages/profile` + 登录资料页 `pages/login`

### 已实现功能

- “我的”页只展示头像、昵称、个性签名
- 私信功能整合进“我的”页，可预览最近会话并进入完整私信页
- 单独的登录与资料页负责初始化账号、选择头像、填写昵称、个性签名、微信号
- 保存资料到 `users`
- 跳转我的发布、学校认证、工具箱

### 为什么这里没有“自动获取真实微信号”

这个项目里我把“微信号”设计成**用户手动填写**。

原因很简单：

- 小程序里能稳定拿到的是：用户身份标识、头像昵称填写能力、以及满足资质时的手机号能力
- 但**个人微信号本身不属于可直接自动读取的公开能力**
- 所以我没有给你做“系统自动读取个人微信号”这种不稳定方案，而是做成了**手动填写 + 私信页一键复制**

这也是校园二手场景里更稳妥、合规的做法。

## 四、云函数说明（按用途拆解）

## 1. `login`

### 作用

获取当前小程序用户的：

- `openid`
- `appid`
- `unionid`

### 原理

小程序调用 `wx.cloud.callFunction` 时，微信会把可信身份注入到云函数上下文，所以这里不需要你自己搭服务器换 code。

---

## 2. `userManager`

### 作用

统一处理用户相关操作：

- `bootstrap`：初始化用户文档
- `getMine`：读取我的资料
- `saveProfile`：保存头像昵称微信号
- `submitSchoolAuth`：提交学校认证申请

### 设计原因

把用户逻辑收敛到一个云函数里，方便后续扩展，例如：

- 拉黑
- 举报
- 收藏
- 用户资料页

---

## 3. `goodsManager`

### 作用

统一处理商品相关操作：

- `create`
- `list`
- `detail`
- `mine`
- `updateStatus`

### 设计原因

前端只负责表单和展示；真正的商品写入由云函数完成。

好处：

- 可控性更强
- 更方便做校验
- 更方便后续加风控、内容审核、敏感词过滤

---

## 4. `messageManager`

### 作用

统一处理聊天能力：

- `ensureConversation`
- `listConversations`
- `listMessages`
- `sendMessage`

### 设计原因

聊天最容易出现权限问题，所以我没有让前端直接写数据库，而是统一交给云函数处理。

这样更容易保证：

- 只有会话成员本人能看消息
- 未读计数统一维护
- 最近消息统一维护

---

## 5. `setup`

### 作用

第一次导入项目时，一键尝试创建所有集合。

### 设计原因

减少你手动点控制台的成本。
